Painel Técnico e Índice IPT

Índice de Prioridade Técnica (IPT)

IPT = (GS + IT + RT) ÷ 3
FÓRMULA REAL: Baseado no código painel.js, o IPT é a média aritmética de três componentes, não uma média ponderada.
Componente Cálculo Escala Significado
GS
Gravidade Sanitária		 Baseado no status final 1-4 ok=1, alerta=2, atencao=3, critico=4
IT
Intensidade Técnica		 IT = ⌈Pontuação ÷ 3⌉ 1-4 Normalização da pontuação (0-12)
RT
Recorrência Temporal		 RT = 1 + (Nº avaliações) 1-4 Persistência do problema (máx 4)

Implementação no Código

// Cálculo do IPT (painel.js - simplificado)
function calcularIPT(escola) {
  // GS: Gravidade Sanitária
  const gs = escola.status === 'ok' ? 1 :
          escola.status === 'alerta' ? 2 :
          escola.status === 'atencao' ? 3 : 4;

  // IT: Intensidade Técnica
  const it = Math.min(4, Math.ceil(escola.pontuacao / 3));

  // RT: Recorrência Temporal
  const rt = Math.min(4, escola.avaliacoes || 1);

  return (gs + it + rt) / 3;
}

Análise Territorial

Criticidade Territorial = Σ(IPTᵢ) ÷ Nᵢ × Área

Princípio de Pareto Espacial: 20% dos territórios concentram 80% da criticidade.

CÁLCULO POR BAIRRO: A criticidade dominante é determinada quando ≥50% das escolas têm mesma classificação. Caso contrário, mostra a classe máxima presente.

Implementação Offline

// Lógica de sincronização (avaliacao.js)
const STORAGE_KEY = "checkinfra_pendentes";

function salvarOffline(dados) {
  const lista = JSON.parse(localStorage.getItem(STORAGE_KEY) || "[]");
  lista.push(dados);
  localStorage.setItem(STORAGE_KEY, JSON.stringify(lista));
}

async function sincronizarOffline() {
  if(!navigator.onLine) return;
  const lista = JSON.parse(localStorage.getItem(STORAGE_KEY) || "[]");
  for(const dados of lista) {
    await db.collection("avaliacoes").doc(dados.id).set({
      ...dados,
      createdAt: firebase.firestore.FieldValue.serverTimestamp()
    });
  }
  localStorage.removeItem(STORAGE_KEY);
}

Service Worker

Funcionalidades do PWA:

  • Cache de recursos estáticos (HTML, CSS, JS)
  • Funcionamento offline completo
  • Atualização em segundo plano
  • Notificações push (planejado)
  • Instalação como aplicativo nativo
EXPERIÊNCIA MOBILE: O sistema funciona como um aplicativo nativo quando instalado na tela inicial do smartphone.

Limitações Offline

1. Armazenamento de Fotos: Imagens em Base64 consomem muito espaço no localStorage
2. Sincronização Parcial: Só sincroniza quando volta online (requer ação do usuário)
3. Recomendação: Para uso intensivo, considerar implementação de IndexedDB (já prototipada em db.js)

Troubleshooting Comum

1

"Não consigo enviar avaliação"

Causa: Sem conexão ou erro no Firebase

Solução: Verifique se está online. Se sim, recarregue a página. Se offline, os dados foram salvos localmente e serão enviados automaticamente.

2

"PDF não está sendo gerado"

Causa: jsPDF não carregado ou erro nas imagens

Solução: Verifique conexão com CDN. Reduza o número/tamanho das fotos.

3

"Mapa não carrega"

Causa: Bloqueio do OpenStreetMap ou erro no Leaflet

Solução: Verifique permissões do navegador. Tente outro navegador.
